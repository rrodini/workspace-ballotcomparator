you specifically need the process handle or more control over process creation, you can use the CreateProcess function. This requires you to find the full path to winword.exe and pass the document path as an argument in the command line string. This method is less flexible than ShellExecute (https://java-native-access.github.io/jna/5.18.1/javadoc/com/sun/jna/platform/win32/ShellAPI.SHELLEXECUTEINFO.html )because the path to Word can change.
